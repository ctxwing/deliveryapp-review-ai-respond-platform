# 프로젝트 헌장 (CONSTITUTION.md)

이 문서는 **AI 이미지 보정 서비스 MVP (Photo Refine Orchestrator)** 개발 프로젝트의 핵심 원칙과 기준을 정의합니다. 모든 개발 및 의사결정은 본 헌장을 최우선으로 따릅니다.

---

## 1. 프로젝트 비전 및 목표
- **비전**: 사용자 업로드 이미지에 대해 AI 분석 및 다중 엔진 보정을 거쳐 최적의 결과물을 제공하는 끊김 없는(Seamless) AI 오케스트레이션 서비스 구축.
- **MVP 목표**: 기능 완성도보다 **사용자 흐름(Flow)의 정상 동작**과 **향후 학습을 위한 정형 데이터 축적**을 최우선으로 함.

## 2. 핵심 원칙 (Core Principles)

### 2.1 비즈니스 및 운영 원칙
- **흐름 중심 개발**: 업로드 → 분석 → 생성 → 비교 → 선택 → 결제 → 제공으로 이어지는 전체 프로세스가 중단 없이 동작해야 함.
- **데이터 자산화**: 사용자의 선택 데이터(RLHF 기초)를 정규화된 DB에 상세히 기록하여 향후 모델 튜닝의 기반을 마련함.
- **비용 최적화**: 개발 단계에서는 Mock API를 활용하여 외부 AI API 호출 비용을 최소화함.

### 2.2 기술 및 엔지니어링 원칙
- **비동기 오케스트레이션**: 실시간 브라우저 대기를 방지하기 위해 **Celery + Redis** 워커 기반의 비동기 처리 구조를 채택함.
- **안정성 및 재시도**: 모든 외부 AI API(Claid, SD, Vision) 호출 실패 시 **최소 2회 재시도**를 수행하며, 최종 실패 시에도 Fallback 정책(기본 점수 부여 등)을 통해 흐름을 유지함.
- **성능 최적화**: 미리보기 및 비교 화면을 위해 저용량 **썸네일 레이어**를 적극 활용함.

### 2.3 디자인 및 UX 원칙
- **Studio Look**: 다크 모드(Dark Mode) 중심의 전문 이미지 편집 스튜디오 분위기를 조성함.
- **시각적 가치 전달**: 비포/애프터(Before/After) 슬라이더를 통해 보정 효과를 직관적으로 비교할 수 있게 함.
- **상태 피드백**: AI 연쇄 작업의 진행 상황(보정 중, 생성 중, 분석 중)을 스켈레톤 UI와 상태 텍스트로 실시간 제공함.

## 3. 기술 스택 (Technology Stack)

| 레이어 | 선정 기술 | 비고 |
| :--- | :--- | :--- |
| **언어/런타임** | **Bun**, Python 3.13+ (uv) | 패키지 관리 및 실행 속도 최적화 |
| **프론트엔드** | **Next.js 16.2.1+ (App Router)** | 사용자 및 관리자 UI 통합 관리 |
| **백엔드** | **FastAPI** | PydanticAI를 통한 데이터 정형화 |
| **BaaS** | **Supabase** | Auth(카카오), PostgreSQL, Storage, Real-time |
| **비동기 작업** | **Celery + Redis** | AI 파이프라인 연쇄 작업 처리 |
| **관리자 패널** | **Refine** | Next.js 통합형 어드민 프레임워크 |
| **AI 엔진** | **Claid, SD (img2img+ControlNet), GPT/Gemini Vision** | 외부 API 조합 |

## 4. 운영 및 개발 규칙

### 4.1 개발 환경 및 디렉토리 구조
- **소스 위치**: 모든 프로젝트 소스는 `./prj_source` 하위에 위치한다.
    - `prj_source/frontend`: Next.js 16.2.1+ (Bun 기반)
    - `prj_source/backend`: FastAPI + Celery Worker 통합 (uv 기반)
- **패키지 매니저**: Node.js는 반드시 `bun`을 사용하며, Python은 `uv` 가상환경을 사용함.
- **데이터베이스**: Drizzle ORM(Frontend) 및 SQLModel(Backend)을 사용하며, Supabase(PostgreSQL + pgvector)를 최종 적용함.
- **알림**: 모든 알림은 `window.alert` 대신 이쁜 **Toast**를 우하단에 표시함.

### 4.2 인프라 및 Docker
- **포트 맵핑**: 
    - Frontend(57104)
    - Backend API(57101)
    - Redis(57102)
    - Postgres(57103)
- **이미지 빌드**: 최신 안정화 버전(Stable Tag)을 명시하며, `Standalone` 빌드 등을 통해 이미지 크기를 최소화함.
- **인증 정보**: 개발/테스트용 계정은 `admin@example.com` / `Admin1234!`를 표준으로 사용함.

### 4.3 Git 정책
- **단일 브랜치**: 로컬과 리모트 모두 오직 `main` 브랜치 하나만 운영함.
- **커밋 메시지**: 반드시 **한글**로 작성하며, 작업 완료 후 Push를 필수 수행함.
