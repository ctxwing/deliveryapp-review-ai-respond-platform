# .specify Workflow 설정 및 패치 노트

## 개요
이 리포지토리의 `.specify` 폴더 내 스크립트가 로컬 프로젝트 규칙(Global Rule)에 맞게 작동하도록 커스텀 수정된 내역을 기록합니다.

---

## 2026-03-21: 워크플로우 브랜치 유효성 검사 패치

**목표:**  
`/speckit.implement` 등 워크플로우 시작 시 기본적으로 기능 브랜치 생성 및 명명 규칙 (`001-feature-name` 등)을 강제하도록 되어있으나, 본 프로젝트는 **오직 `main` 브랜치에서만 모든 작업을 수행**해야 한다는 전역 규칙을 준수하기 위해 스크립트를 수정함.

**수정 파일:**  
- `.specify/scripts/bash/common.sh`

**주요 변경 사항:**
1. **`main` 브랜치 우회 허용:**
   - `check_feature_branch()` 함수에서 현재 브랜치가 `main` 또는 `master`일 때, 에러 없이 `return 0`으로 통과시키도록 로직을 추가.
2. **`main` 브랜치에서의 디렉토리 매핑:**
   - `get_current_branch()` 함수에서 `main` 브랜치일 경우에는 브랜치명 기반 매핑 로직을 무시하고, `specs/` 폴더 내에 있는 '가장 숫자가 높은' 최신 기능 디렉토리를 찾아 반환하도록 수정.
3. **디렉토리 정규식 변경 (`[0-9]{3}` → `[0-9]+`):**
   - 기존의 세 자리 숫자 강제 `^[0-9]{3}-` 규칙을 폐기하고 자리수에 상관없는 `^[0-9]+-` 패턴으로 변경하여, `1-proch-aep-analysis`와 같은 단일 숫자 시작 디렉토리도 정상 인식하도록 개선.

**결과:**  
이제부터 개발자는 새로운 브랜치를 생성하지 않고 `main` 브랜치에 있는 상태 그대로 `/speckit.implement` 등의 자동화 워크플로우 명령어를 에러 없이 실행할 수 있습니다.
